05TradingView indicator — live confirmation
Same swing-high/low, ATR, volume-ratio and proximity logic as the scanner above, ported to Pine Script v5, so you can watch for the actual intraday confirmation (price crossing entry with volume) instead of relying on EOD data alone.
  1. Open the chart on TradingView for the stock you want → Pine Editor tab at the bottom.
  2. Delete the placeholder script, paste the code below, click Add to chart.
  3. Set the inputs (lookback, proximity %, min vol ratio, RR target, T1/T2 %) to match whatever you used in Step 02 here. These are pulled from Daily data internally, so the setup itself won't change no matter what chart timeframe you're viewing.
  4. Chart timeframe choice: use the Daily chart to see the full table and the once-per-day setup triangle clearly. Switch to a 5-min or 15-min chart on the same script to watch the "entry confirmed" circle — it only fires the moment price actually trades through the Daily entry level, so this is your live wait-for-confirmation signal.
  5. Right-click the chart → Add alert → condition = this indicator → choose "Long setup flagged" / "Long entry confirmed" (or the short equivalents) → "Long/short setup flagged" should trigger on bar close (Daily), and "entry confirmed" on once per bar close of whichever intraday timeframe you're watching.
  6. If your chart already has several other indicators/studies (Supertrend, MACD, RSI signals, etc.), their markers will show up alongside this one — temporarily hide the others if the chart looks cluttered, so you're only reading this indicator's triangles/circles.
  7. This is a starting point, not a certified match to the scanner above — TradingView's live/adjusted prices can differ slightly from NSE bhav copy (corporate actions, data vendor). Cross-check before placing size.

      

    

  


  
  

    
06Live prices — local proxy (recommended)

    
This page has no server, and browsers block direct cross-origin requests to Yahoo Finance from a page like this one (CORS). Without the proxy below, the "Refresh live prices" buttons will try Yahoo directly, then a public CORS-bypass proxy — both can be blocked, rate-limited, or slow. Running this once locally makes it reliable — whether you're opening this file directly or hosting it on GitHub Pages. One browser detail: if this page is served over HTTPS (like GitHub Pages) and you're on Chrome, reaching a proxy on localhost triggers a one-time "Allow local network access?" popup — click Allow. This is Chrome protecting your network, not a bug in this tool.
